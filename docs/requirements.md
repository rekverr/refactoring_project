# Вимоги та симуляція

## Опис проблеми
Система моделює роботу бібліотеки без зовнішньої бази даних. Дані зберігаються в In-Memory репозиторіях, щоб проєкт легко запускався у CI/CD та перевірявся статичним аналізатором.

Ключові логічні проблеми:
- видача книги тільки активному користувачу;
- контроль ліміту активних позик;
- повернення книги та нарахування штрафу;
- пріоритетність черги резервування;
- блокування користувача;
- сповіщення користувача, коли книга стає доступною.

## Актори
- **Читач** — бере книги, повертає книги, резервує книги.
- **Бібліотекар** — блокує/розблоковує читачів, контролює повернення та резервування.

## Use Cases
1. Читач бере доступну книгу.
2. Читач повертає книгу.
3. Система нараховує штраф за прострочення.
4. Читач резервує недоступну книгу.
5. Система визначає наступного читача у черзі резервування.
6. Система надсилає сповіщення про доступність книги.
7. Бібліотекар блокує користувача.
